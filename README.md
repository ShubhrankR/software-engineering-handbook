# Software Engineering Handbook

> A personal knowledge base documenting my journey to becoming a better Software Engineer.

## Why this repository?

I started this repository while working as a Senior Software Engineer at VISA.

After several unsuccessful interview attempts, I realized that my biggest weakness wasn't a framework—it was my understanding of software engineering fundamentals.

Instead of collecting random notes, this repository will become my long-term engineering handbook.

The goal is to deeply understand how software works, build real-world projects, prepare for technical interviews, and continuously grow as a Software Engineer.

---

## My Goal

Become a Software Engineer capable of building high-quality products, succeeding in interviews at top product companies, and effectively leveraging AI in software development.

---

## Learning Principles

* Understand concepts instead of memorizing them.
* Learn by building and experimenting.
* Document everything in my own words.
* Focus on fundamentals before frameworks.
* Continuously improve through interviews and projects.
